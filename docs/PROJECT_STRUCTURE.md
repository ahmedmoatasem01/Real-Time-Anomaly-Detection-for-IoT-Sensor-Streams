# Project Structure

This document explains what each top-level folder is for, where generated
artifacts live, and which scripts do the real work. It reflects the repository
as of the 2026-07-06 cleanup pass (see `PROJECT_SCAN_AND_CLEANUP_PLAN.md` for
the audit that drove it).

## Root layout

```
README.md                          Project overview, setup, API docs, demo walkthrough
MASTER_ROADMAP_MULTIMODAL_PLATFORM.md  Current master build/roadmap document
PROJECT_SCAN_AND_CLEANUP_PLAN.md   Active audit + cleanup tracking doc (see docs/archive/ for older, superseded plans)
requirements.txt                   Python dependencies
pyproject.toml                     pytest config (pythonpath, testpaths) + pyright config
docker-compose.yml                 api + frontend service definitions
.gitignore / .dockerignore         VCS / image build exclusions
conftest.py                        Ensures `src` is importable when running bare `pytest`
demo_loop.py                       Manual demo helper script
```

## `src/` — backend (FastAPI + ML)

| Folder | Purpose |
|---|---|
| `src/api/` | FastAPI app (`main.py`), request/response schemas, the live `InferenceService`, and the vibration/image/asset routers (`vibration_router.py`, `image_router.py`, `asset_router.py`) |
| `src/data/` | CSV loading and preprocessing for the NAB temperature stream |
| `src/features/` | Rolling-window feature engineering (mean, std, EWMA, lag, z-score, FFT) |
| `src/models/` | One training script per algorithm (`train_isolation_forest.py`, `train_lof.py`, `train_elliptic_envelope.py`, `train_one_class_svm.py`, `train_river_online.py`, `train_lstm_autoencoder.py`, `train_baseline.py`), plus `evaluate_all.py` (the one true evaluation entrypoint — produces `reports/evaluation_results.csv`) |
| `src/registry/` | `model_registry.py` — reads/writes `models/model_registry.json`, the single source of truth for which model is in production |
| `src/drift/` | PSI / mean-shift drift detection (`drift_detector.py`, `drift_service.py`) — writes `reports/drift_status.json` / `drift_history.json` |
| `src/retraining/` | `retrain_pipeline.py` — exports recent readings, retrains, evaluates, and promotes a new production model; triggered by `POST /models/retrain` |
| `src/synthetic/` | Synthetic fault injection (spikes, drift, freeze) used by the Demo Control Panel, plus synthetic vibration/vision data generators |
| `src/vibration/` | NASA Bearing (vibration) modality: loader, FFT features, Isolation Forest training |
| `src/image/` | MVTec-style vision modality: ResNet50 embeddings, Isolation Forest training |
| `src/database/` | SQLAlchemy models (`Reading`, `Alert`, `ModelRun`, `Asset`) and session management |
| `src/reports/` | `incident_report.py` — generates the per-alert PDF incident report |
| `src/streaming/` | `stream_simulator.py` — multi-threaded historical-data replay used as the "live" feed |
| `src/experiments/` | Append-only experiment tracker (`reports/experiments.json`) |
| `src/utils/` | Settings (`config.py`, env-var driven) and logging setup |

## `tests/` — pytest suite

One file per concern: `test_preprocessing`, `test_features`, `test_leakage`,
`test_model`, `test_stream`, `test_database`, `test_api`, `test_drift`,
`test_fault_injection`, `test_incident_report`, `test_alert_lifecycle`. Run
with `pytest -q` or `python -m pytest` from the repo root.

## `models/` — trained artifacts

`*.pkl` (scikit-learn/River models), `*.pth` (raw LSTM state dict, not
loaded at inference time — the `.pkl` wrapper is), `scaler.pkl`,
`feature_columns.json` / `feature_stats.json`, per-model `threshold_*.json`
(validation-split thresholds), and `model_registry.json` (the only model
artifact that's actually committed to git — everything else is generated
locally by the training scripts and gitignored).

## `reports/` — generated outputs

`evaluation_results.csv` and `model_comparison.json` (test-split metrics for
all 7 models, produced by `evaluate_all.py`), `experiments.json` (training
run history), `drift_status.json` / `drift_history.json` (live drift-check
results), and `reports/figures/*.png` (per-model PR/ROC/confusion plots —
gitignored, regenerated by `evaluate_all.py`).

## `docs/` — documentation

| Folder | Purpose |
|---|---|
| `docs/final_report/` | The academic final report (`FINAL_PROJECT_REPORT.md`/`.pdf`), its section drafts (`sections/`), and diagrams (`figures/`) |
| `docs/final_report/figures/` | Architecture/ERD/sequence diagrams (mermaid-sourced `.mmd` + rendered `.png`) |
| `docs/architecture/` | Reserved for architecture-specific documentation as the platform grows |
| `docs/model_cards/` | One markdown card per model (intended use, data, features, metrics, limitations, latency) |
| `docs/archive/` | Superseded planning/roadmap documents kept for history — not active instructions |

## `frontend/` — React SPA

`src/pages/` holds one file per route (Overview, Live Monitor, Alert Center,
Asset Center, Data Explorer, Demo Panel, Experiment Log, Model Lab,
Retraining Center, System Health, Vibration Lab, Vision Lab), all wired in
`src/App.tsx`. `src/lib/api.ts` and `ws.ts` are the REST/WebSocket clients
(note: the vibration/image/asset pages currently call their endpoints with
inline `fetch`/`WebSocket` calls rather than through this shared client —
see `PROJECT_SCAN_AND_CLEANUP_PLAN.md` §3). `src/components/ui/` holds
shadcn-style primitives (`button`, `card`, `badge`).

## `scripts/`

`demo_reset.py` (resets the demo database), `generate_diagrams.py` (renders
the mermaid `.mmd` files in `docs/final_report/figures/`),
`generate_final_report.py` (compiles the final report markdown to PDF).

## `docker/`

`Dockerfile` for the API image; the frontend has its own `Dockerfile` under
`frontend/`. Both are wired together by the root `docker-compose.yml`.
